# 에이전트 템플릿 가이드

## 목적

`.harness/60_templates/agents/`는 하네스 기준 에이전트 마스터 템플릿을 보관해요.

## 형식 매핑 규칙

| 형식 | 용도 |
|---|---|
| `md` | Claude 또는 범용 LLM 템플릿 |
| `toml` | Codex 전용 템플릿 |

- 같은 역할이라도 런타임 형식은 다를 수 있어요.
- 다만 역할 의미, 책임 범위, 판정 기준은 마스터 기준으로 맞춰요.

## 템플릿 분류

| 분류 | 설명 |
|---|---|
| guardian | 규칙 위반, 범위 이탈, 핸드오프 품질을 검토해요 |
| feature | 구현 전담 에이전트 베이스예요 |
| planner | 설계와 구현 계획 전담 베이스예요 |
| utility | 커밋, 이슈 같은 운영 액션 에이전트 베이스예요 |
| reviewer | 리뷰 전담 베이스예요 |
| orchestrator | 라우팅과 계획 조정 베이스예요 |

## 새 에이전트 추가 체크리스트

- [ ] `.harness/60_templates/agents/`에 대응 base 템플릿이 있어요.
- [ ] 대상 LLM 레이어(`.claude/agents/`, `.codex/agents/`)에 파생본을 만들었어요.
- [ ] guardian 계열이면 `extends: guardian-base`를 넣었어요.
- [ ] 관련 README와 진입 문서의 Agents 목록을 갱신했어요.
- [ ] 하네스 정책 또는 bridge 문서에서 참조가 필요한지 확인했어요.

## 에이전트 템플릿 동기화 체크리스트

- [ ] 마스터 변경 후 파생본 CORE 영역을 동기화했어요.
- [ ] ENV 값은 각 런타임에 맞게 유지했어요.
- [ ] 파생본 의미가 마스터와 충돌하지 않아요.
