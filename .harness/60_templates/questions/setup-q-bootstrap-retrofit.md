# Setup Q&A — Bootstrap or Retrofit

## 질문

- 지금은 새 환경 셋업인가요, 기존 환경 보강인가요?

## 선택지

| 선택 | 판단 기준 | 다음 읽기 |
|---|---|---|
| `bootstrap` | `.claude/`, `.codex/`, `.ai/` 자산이 아직 없거나 매우 얇아요 | `.harness/70_init-packs/COMMON_INIT_SEQUENCE.md` |
| `retrofit` | 이미 운영 중인 자산이 있고 의미 보존이 중요해요 | `.harness/10_bootstrap/retrofit-architecture.md` |

## 결정 메모

- `retrofit`은 기존 의미를 깨지 않는 보강이 우선이에요.
- `bootstrap`은 기본 구조를 깨끗하게 심는 데 집중해요.
