# Setup Q&A — Verify LLM

## 질문

- 검증(Verify)을 담당할 LLM은 무엇인가요?

## 선택지

| 선택 | 의미 | 권장 상황 |
|---|---|---|
| `Work LLM과 동일` | 구현과 검증을 한 실행면에서 처리해요 | 기본값, 속도 우선 |
| `별도 LLM` | 구현과 검증 시각을 분리해요 | 고위험 변경, critique 강화 |

## 결정 메모

- 별도 LLM을 고르면 evidence 기준을 더 엄격히 정해요.
- 같은 LLM을 고르면 최종 판정자(Final Check)를 분리하는 것이 좋아요.
