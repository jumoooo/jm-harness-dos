# Contamination Analysis Policy

## 목적

기존 환경에 하네스를 적용할 때 오염 가능성을 어떻게 판단하고 보고할지 정의해요.

## 오염 후보 예시

- 기존 규칙 파일 덮어쓰기
- 기존 agent 책임 변경
- 기존 템플릿 무효화
- 기존 로컬 진입 명령의 의미 변경
- 기존 산출물 경로 충돌

## 현재 저장소의 즉시 관찰

- `.claude/`와 `CLAUDE.md`가 이미 Claude 전용 진입 구조를 설명하고 있어요.
- 따라서 Claude 대상 bootstrap 또는 retrofit은 경로 충돌보다 "의미 충돌"을 먼저 봐야 해요.
- 이번 phase에서는 문서만 추가하므로 직접 오염은 만들지 않아요.

## 사용자 보고 필수 항목

- 무엇이 무엇으로 바뀌는지
- 어떤 파일, 규칙, 에이전트가 영향을 받는지
- side effect
- 대안별 점수
- 추천안
- 하네스 방향성과의 거리

## 보고 기준

- 기존 `.claude/` 명령 의미를 바꾸면 반드시 사전 보고해요.
- 기존 `.ai/` 공유 작업면 계약을 바꾸면 반드시 사전 보고해요.
- 삭제, 이동, 덮어쓰기가 포함되면 승인 없이 진행하지 않아요.
