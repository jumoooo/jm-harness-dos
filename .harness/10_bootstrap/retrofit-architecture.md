# Retrofit Architecture

## 목적

이미 구축된 LLM 환경을 하네스 기준으로 재구축하거나 보강하는 절차를 정의해요.

## 기본 흐름

1. 기존 환경 인벤토리 분석
2. 하네스 기준과 gap 비교
3. contamination 후보 추출
4. 점수표 작성
5. 추천안 도출
6. 사용자 승인
7. phase별 재구축 진행

## 반드시 포함할 분석

- 현재 명령, 에이전트, 메모리 구조
- 기존 규칙과 하네스 규칙 충돌
- 삭제, 이동, 덮어쓰기 위험
- side effect
- 보존 가능한 부분과 폐기 또는 대체가 필요한 부분

## 현재 저장소에 대한 주의

- `.claude/`는 이미 명령과 에이전트 구조를 갖고 있어요.
- 따라서 Claude 계열 적용은 기본적으로 retrofit 후보로 먼저 봐요.
- 이번 phase에서는 실제 재구축을 시작하지 않아요.
