# Init Entry

## 목적

하네스 셋업 요청이 들어왔을 때,
무엇을 진입 신호로 보고 어떤 순서로 해석할지 고정해요.

## 진입 신호

- `.harness/... init 해줘`
- `.harness/BOOTSTRAP_GUIDE.md 기준으로 셋업해줘`
- 특정 init pack 경로를 태그해서 init 요청

## 해석 규칙

1. 먼저 `00_foundation/`을 읽어요.
2. 현재 요청이 bootstrap인지 retrofit인지 판정해요.
3. 단일 LLM으로 충분한지 먼저 확인해요.
4. 복수 LLM이 필요해 보이면 사용자 질문 항목으로 올려요.
5. contamination 가능성을 먼저 정리해요.
6. high-impact 변경이면 승인 전에는 실행하지 않아요.

## bootstrap 판정 기준

- 새 환경이거나 하네스 계층이 아직 없는 경우
- 하네스 진입 파일과 표준 폴더가 아직 없는 경우
- 기존 도구 자산은 있어도 하네스 표준 연결이 없는 경우

## retrofit 판정 기준

- 기존 LLM 전용 자산이 이미 운영 중인 경우
- 명령, 에이전트, 템플릿, 규칙이 이미 존재하는 경우
- 하네스 기준으로 보강하거나 재배치가 필요한 경우

## 출력 기대

- 현재 환경 분석
- bootstrap 또는 retrofit 판정
- 사용자 확인 필요 항목
- 추천 진행 경로

## 메모

> ⚠️ init 요청이 들어와도 바로 수정부터 시작하지 않아요.
> 먼저 판정과 영향 분석을 끝내요.
