# Token Efficiency Principles

## 기본 원칙

- 긴 대화에 상태를 누적하지 않아요.
- 상태는 문서와 링크로 외부화해요.
- 정적 규약은 하네스 문서로 고정하고, 작업별 가변 정보는 task-order로 분리해요.
- 구현 LLM은 필요한 폴더와 문서만 읽어요.
- 상세 변경 내역과 critique 결과는 별도 산출물로 분리해요.

## 실무 규칙

- 상위 인덱스 문서는 짧게 유지해요.
- 세부 규칙은 역할별 폴더에 나눠요.
- 한 번에 한 문서에 모든 걸 넣지 않아요.
- 구현 지시는 phase별로 나눠요.

## 메모

> 📎 foundation은 규약만 담고, 절차는 다음 번호 폴더로 넘겨요.
